# MPU6050 library design

## Scope

Create a Jaculus TypeScript library and JacLy blocks for the MPU6050 accelerometer and gyroscope. The default configuration must work without extra setup while exposing both supported I2C addresses and every accelerometer and gyroscope range.

## TypeScript API

- `Vector3` represents `{ x, y, z }` numeric readings.
- `AccelRange` exposes 2 g, 4 g, 8 g, and 16 g ranges.
- `GyroRange` exposes 250, 500, 1000, and 2000 degrees-per-second ranges.
- `MPU6050Options` optionally selects the address and both ranges.
- `new MPU6050(i2c, options?)` defaults to address `0x68`, 2 g, and 250 degrees per second.
- Construction wakes the sensor, selects the gyroscope X-axis PLL clock, and applies both ranges.
- `readAcceleration()`, `readGyroscope()`, and `readTemperature()` return g, degrees per second, and degrees Celsius respectively.
- Range setters preserve unrelated register bits and update the cached range. Range getters return the active cached selection.
- Invalid addresses or enum values throw a descriptive `RangeError`. I2C errors propagate unchanged.

## JacLy blocks

The `mpu6050` peripherals category contains the shared I2C constructor, an MPU6050 constructor with address and range dropdowns, statement blocks for changing each range, value blocks for acceleration, gyroscope, and temperature, and a value block that extracts `x`, `y`, or `z` from a `Vector3`.

English and Czech translations cover the category, messages, tooltips, and dropdown labels. Generated code imports `MPU6050`, `AccelRange`, and `GyroRange` from `mpu6050`.

## Verification

Run the package build, parse every JSON block/translation file, and inspect the generated declarations. Hardware behavior follows the official MPU-6000/MPU-6050 register map; physical validation requires a connected sensor and is outside this workspace check.
